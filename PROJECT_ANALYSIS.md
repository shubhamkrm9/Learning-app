# Project Analysis: Learning SDK

This document provides a comprehensive overview of the **Learning SDK** project. It is structured to help you understand the architecture, API flow, UI/UX structure, and testing methodology so you can effectively explain it to your team lead and other developers.

---

## 1. Project Architecture & Flow

The project is a **React Native Library (SDK)** built using a monorepo structure (managed via Yarn workspaces and `react-native-builder-bob`). It is designed to be injected into any host React Native application to provide a full-fledged "Learning Module" experience.

### Directory Structure
- `src/`: The core source code of the SDK.
- `example/`: A sample host application that imports and runs the SDK for testing and development.
- `lib/`: The compiled, production-ready output of the SDK generated by `bob build`.
- `android/` & `ios/`: Native configurations for the library.

### Key Components & Files

#### SDK Entry Points (`src/sdk/`)
- `LearningSDK.ts`: The imperative controller (Singleton). Provides methods like `initialize()`, `launch()`, `pause()`, and `destroy()`. It uses `DeviceEventEmitter` to broadcast events to the UI.
- `LearningSDKView.tsx`: The main root UI component. It listens to events from `LearningSDK.ts`. When `launch()` is called, it renders the entire Learning SDK flow as a full-screen view overlaid on the host app.
- `SDKProvider.tsx`: Wraps the SDK in necessary contexts: Redux Store (`Provider`), Safe Area Provider, and a custom `SDKContext`.

#### Navigation (`src/navigation/`)
- `RootNavigator.tsx`: Uses `@react-navigation/native-stack`. It is completely independent (`independent={true}`) of the host application's navigation.
- **Screens**:
  - `Dashboard`: The main entry list of modules.
  - `ModuleDetail`: Detailed view of a specific module.
  - `VideoPlayer`: Custom video playback screen.
  - `Assessment` & `AssessmentResult`: Quiz and results screens.
  - `ModuleCompletion` & `Certificate`: Final completion screens.

#### State Management & API (`src/store/` & `src/api/`)
- The SDK uses **Redux Toolkit (RTK)** and **RTK Query** for state and API management.
- `store/index.ts`: Creates a dynamically instanced Redux store to avoid clashing with the host app's store.
- `api/baseApi.ts`: The central RTK Query configuration. It dynamically injects the `baseUrl` and `accessToken` (provided during `LearningSDK.initialize()`) into every API request header.
- `api/moduleApi.ts`, `api/assessmentApi.ts`: Specific endpoints injecting into `baseApi`.

#### UI/UX & Theming (`src/theme/` & `src/components/`)
- The UI uses a strict design system defined in `src/theme/` (`colors.ts`, `spacing.ts`, `typography.ts`).
- `src/components/`: Reusable, stateless UI components (e.g., `Button`, `Card`, `ProgressBar`, `ErrorState`, `EmptyState`) to maintain visual consistency.

### Data Flow (How it Works)
1. **Initialization**: The host app calls `LearningSDK.initialize({ accessToken, baseUrl })`.
2. **Launch**: The host app calls `LearningSDK.launch()`.
3. **Event Emitted**: `LearningSDK` emits a `LAUNCH` event.
4. **UI Render**: `LearningSDKView` catches the event, sets `isVisible` to true, and mounts `RootNavigator` over the host app.
5. **Data Fetching**: The `DashboardScreen` mounts, triggering an RTK Query hook. The query uses the injected `baseUrl` and `accessToken` to fetch learning modules.
6. **Exit**: The user finishes or closes the SDK, `LearningSDK.destroy()` is called, tearing down the UI and store.

## 2. What is the `example` folder?

The `example` folder is a completely independent, barebones React Native application nested inside the project.

**Its purpose:**
It acts as the "Host Application" during development. Since an SDK cannot run on a phone by itself (it's just a library of code), the `example` app is used to import the SDK, run it, and test it on a real device or emulator.

If you look at `example/src/App.tsx`, it imports `LearningSDK` and `LearningSDKView` from `react-native-learning-sdk`, initializes it with dummy data, and provides a button to launch it.

### Why do changes in the SDK reflect in the `example` app automatically?
This repository uses **Yarn Workspaces** and **Metro Bundler**. In `example/package.json`, the SDK is linked locally. When you run the `example` app, the Metro bundler is configured (via `react-native-monorepo-config` or custom metro config) to watch the `../src` directory.
Therefore, any time you save a file in the SDK's `src` folder, the Metro bundler instantly detects the change, recompiles it, and triggers Fast Refresh on your phone running the `example` app. You don't need to manually rebuild or change the example app to see your SDK updates!

## 3. How to Change a Feature and Test it on Your Phone

**Scenario**: You want to change the color of the primary button in the SDK.

1. **Make the Change**:
   - Open `src/theme/colors.ts`.
   - Modify the `primary` color hex code.
   - Save the file.
2. **Test on Phone**:
   - Open a terminal in the root directory.
   - Run the example app on your Android device by executing:
     ```bash
     yarn example android
     ```
   - *Note: Make sure your physical Android device is connected via USB with USB Debugging enabled, or you have an Android Emulator running.*
3. **Verify**:
   - The app will build and launch on your phone.
   - Click "Launch Learning Module".
   - You will instantly see the new button color. If you change the code again while the app is open, it will hot-reload automatically.

## 4. Developer Guide: How to Import and Use this SDK

This guide is for other developers who want to integrate your SDK into their React Native applications.

### Step 1: Installation
Install the SDK via npm or yarn:
```bash
yarn add react-native-learning-sdk
```

*Note: Since the SDK has peer dependencies, ensure the host app has `react-native-safe-area-context`, `react-native-screens`, `react-native-video`, and `react-native-gesture-handler` installed.*

### Step 2: Mount the View Component
The SDK provides a `LearningSDKView` that renders the UI. It operates like a Modal, so it should be placed at the root level of your application (e.g., inside your `App.tsx` or main layout).

```tsx
// App.tsx
import React, { useEffect } from 'react';
import { View, Button } from 'react-native';
import { LearningSDK, LearningSDKView } from 'react-native-learning-sdk';

export default function App() {
  // 1. Initialize the SDK with user credentials and your backend URL
  useEffect(() => {
    LearningSDK.initialize({
      accessToken: 'user_auth_token_here',
      baseUrl: 'https://api.yourbackend.com',
      onEvent: (eventName, payload) => {
        console.log("SDK Event:", eventName, payload);
      }
    });
   
    // Cleanup on unmount
    return () => LearningSDK.destroy();
  }, []);

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
      {/* 2. Trigger the SDK to open */}
      <Button
        title="Open Training Modules"
        onPress={() => LearningSDK.launch()}
      />
     
      {/* 3. Mount the SDK View at the root level */}
      <LearningSDKView />
    </View>
  );
}
```

### SDK API Reference

The `LearningSDK` object provides the following methods:

- `LearningSDK.initialize(config: SDKConfig)`: Must be called before launching. Sets up the auth token and backend URL.
- `LearningSDK.launch()`: Opens the SDK UI overlay over the host app.
- `LearningSDK.pause()`: Pauses background activities (like video playback) when the host app goes into the background.
- `LearningSDK.resume()`: Resumes background activities.
- `LearningSDK.destroy()`: Closes the SDK and clears the internal Redux state and configurations.
- `LearningSDK.logout()`: Specifically handles user logout scenarios, clearing data and closing the UI.

# Changelog

This document summarizes all the modifications and fixes applied to the `react-native-learning-sdk` to ensure flawless integration with the `FinfinityNativeMobile` consumer application.

## 1. Dependency Downgrade & Alignment
To match the host application's environment, the following dependencies were downgraded or removed:
- **React Native**: Downgraded from `0.85.x` to `0.84.1`.
- **React**: Downgraded to `19.2.3`.
- **React Native Gesture Handler**: Downgraded to `^2.30.0`.
- **React Native Reanimated**: Completely removed to reduce integration complexity, as the current animations could be handled natively or weren't strictly necessary.
- **Babel & ESLint Configs**: Downgraded `@react-native/babel-preset` and `@react-native/eslint-config` to `0.84.1`.

## 2. ESLint & TypeScript Fixes
- Reverted the modern ESLint flat config (`eslint.config.js`) back to the legacy `.eslintrc.js` to restore compatibility with older ESLint plugins used in React Native 0.84.
- Fixed unused `err` variables across multiple screens:
  - `src/screens/AssessmentResult/AssessmentResultScreen.tsx`
  - `src/screens/Assessment/AssessmentScreen.tsx`
  - `src/screens/Certificate/CertificateScreen.tsx`

## 3. Android Build & NDK Configuration
- **Compile SDK Version**: Bumped `compileSdkVersion` and `targetSdkVersion` to `36` in `example/android/build.gradle` to resolve compilation errors from `androidx.core:core:1.17.0` (which is required by some native community modules).
- **NDK Version Stabilization**:
  - Locked the NDK version to **26.1.10909125** in `example/android/build.gradle` to avoid ABI mismatch errors (`__cxa_guard_acquire`) that occurred when attempting to use NDK 27 with React Native 0.84.1's pre-built AARs.
- **C++ Header Patch**:
  - Manually patched a bug in React Native 0.84.1's `graphicsConversions.h` (located in the `.gradle` cache) that incorrectly used the C++20 `std::format` feature. Replaced it with a C++17 compatible `std::snprintf` polyfill to allow successful compilation under NDK 26.

## 4. UI & Navigation Architecture Fixes
- **Fabric Architecture Crash Fix**:
  - Fixed a critical crash (`ScreenContainer is not attached under ReactRootView`) that occurred upon clicking the "Launch Learning Module" button.
  - **Resolution**: React Navigation's `react-native-screens` fails to attach properly when nested inside a React Native `<Modal>` under the New Architecture (Fabric). Modified `src/sdk/LearningSDKView.tsx` by replacing the `<Modal>` component with a full-screen absolute `<View>` (`StyleSheet.absoluteFill`). This ensures the navigation stack remains correctly anchored to the primary `ReactRootView` while preserving the desired overlay appearance.

## Appendix: Codebase Deep Dive

To provide further clarity on the internal structure, here is a detailed breakdown of the internal source folders based on the actual codebase structure:

### `src/components/` (Reusable UI Elements)
- **`Assessment/`, `Button/`, `Card/`, `EmptyState/`, `ErrorState/`, `Header/`, `Loader/`, `ProgressBar/`**: These folders contain modular, stateless components. By isolating components like `ProgressBar` and `Card`, the SDK ensures that the UI remains consistent across all learning modules and quizzes.

### `src/screens/` (View Layer)
- **`Dashboard/`**: Renders the main entry view displaying all available learning modules.
- **`ModuleDetail/`**: Shows the syllabus and details of a specific module.
- **`Video/`**: A custom video playback screen for rendering training videos.
- **`Assessment/` & `AssessmentResult/`**: Handles the quiz UI (multiple choice questions) and the score report.
- **`Certificate/` & `ModuleCompletion/`**: Screens shown upon successful completion of a module.

### `src/api/` (Network Layer)
- **`baseApi.ts`**: The core API configuration. It dynamically intercepts requests to inject the `X-Base-URL` and `Authorization` headers based on what was passed to `LearningSDK.initialize()`.
- **`moduleApi.ts`**: Fetches the lists of modules and their details.
- **`assessmentApi.ts`**: Handles fetching quiz questions and submitting user answers.
- **`progressApi.ts`**: Syncs the user's video watch progress to the backend.
- **`certificateApi.ts`**: Fetches the completion certificate metadata.

### `src/theme/` (Design System)
- **`colors.ts`**: Defines the color palette (e.g., `primary`, `background`, `text`, `error`).
- **`spacing.ts`**: Defines standard padding and margin values to ensure uniform layout spacing.
- **`typography.ts`**: Defines font sizes, weights, and line heights.
